# 第一阶段·第一层：核心定位（必答，3问）

> 本层问题为必答项，确定小说的基本骨架。每个问题提供热门选项 + 自由输入。

---

## Q1：题材与创意概要

使用 `AskUserQuestion` 工具询问：

```
Question: 你想要创作什么题材的小说？
Options:
- 悬疑推理（侦探、破案、解谜）
- 言情（现代都市 / 古代宫廷 / 穿越时空）
- 奇幻玄幻（魔法、异世界、修真）
- 科幻未来（科技、太空、末世）
- 武侠仙侠 / 历史架空（江湖、门派、朝堂权谋）
- 都市现实 / 成长（生活、职场、社会）
- ⭐自由描述（我有明确的想法，让我自己说）
```

> **偏好集成**：如果 `favoriteGenres` 中有匹配项，将对应选项排在最前面并标记⭐。

**记录**：`题材 = [用户选择]`
**记录**：`创意概要 = [用户在"自由描述"中已包含足够信息]`

---

## Q2：主角与关系网络

使用 `AskUserQuestion` 工具询问：

```
Question: 主角是什么设定？
Options:
- 男性主角（独角戏）
- 女性主角（独角戏）
- 双主角（男女双线 / 对手戏）
- 群像戏（多线叙事）
- ⭐自由描述（我已想好角色设定）
```

> **偏好集成**：如果 `preferredProtagonist` 有值，标记对应选项为⭐。

**记录**：`主角类型 = [用户选择]`

**追问1 — 职业/身份**（如需）：
```
Question: 主角的职业或身份是？（简短回答即可）
Options:
- [基于题材推荐的5~7个常见职业]
- 自由输入
```

**记录**：`职业/身份 = [用户回答]`

**追问2 — 核心性格**：
```
Question: 主角的核心性格是？
Options:
- 热血正义（积极、勇敢、有担当）
- 冷静智慧（理性、谋略、高智商）
- 温暖治愈（善良、温柔、有同理心）
- 高冷孤傲（冷漠、独立、强大）
- 阴暗腹黑（心机、算计、复仇）
- 成长逆袭（从弱到强、打脸升级）
- 自由描述
```

**记录**：`核心性格 = [用户回答]`

**追问3 — 关系网络**（可选引导）：
> 如果用户未主动提及，轻量提示："有没有已经想好的关键配角？比如对手、盟友、恋人？（可跳过）"

**记录**：`关键配角 = [用户回答 / 无]`

## Q3：核心冲突与驱动力

使用 `AskUserQuestion` 工具询问：

```
Question: 小说的核心冲突是什么？
Options:
- 生死存亡（生存危机、逃出生天）
- 查明真相（寻找答案、揭露秘密）
- 爱情阻碍（追求真爱、克服阻碍）
- 复仇雪恨（复仇计划、伸张正义）
- 权力争夺（竞争上位、资源争夺）
- 成长突破（自我突破、实现价值）
- 守护保护（守护重要的人或事）
- 自由描述
```

**记录**：`核心冲突 = [用户选择]`

**追问 — 驱动力**：
```
Question: 是什么推动主角不断向前？主角的内在驱动力是什么？
Options:
- 复仇（为某人/某事而战）
- 爱情（为所爱之人）
- 责任/使命（不得不做）
- 好奇心/求知欲（想知道真相）
- 生存（活下去）
- 野心（想要得到某样东西）
- 自由描述
```

**记录**：`驱动力 = [用户回答]`

## 第一层完成 → 摘要展示与过渡

Q1-Q3 全部完成后，执行以下两步：

### 1. 展示已明确信息

向用户展示：

```markdown
## 已收集信息

📖 **题材**：[题材] — [创意概要]
🧑 **主角**：[主角类型] | [职业/身份] | [核心性格]
💥 **冲突**：[核心冲突] | 驱动力：[驱动力]
```

### 2. 静默更新偏好

将第一层的回答同步到 `user-preferences.json`（不打扰用户）。

偏好数据结构与更新规则详见 → [shared-infrastructure.md](shared-infrastructure.md)

一句话引导进入深度定制：

> "核心定位已完成！接下来是深度定制环节（世界观、视角基调、主题、读者定位、章节数量等），每个问题都可以跳过或随机生成。准备好了吗？"

→ 进入 **第二层：深度定制与创作规格（Q4-Q8）**，详见 [phase1-layer2-customize.md](phase1-layer2-customize.md)
